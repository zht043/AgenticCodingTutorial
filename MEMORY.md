# Project Memory: Agentic Coding Tutorial

> 用来记录本仓库在写作和结构设计上的长期决策与偏好，方便 Agent / 未来维护者快速对齐上下文。

## 写作目标

- 服务对象：已经会写代码，但对 Agent / AI 编程还不熟悉的中文开发者。
- 平台形态：以 GitHub 仓库为主要发布载体，结构清晰、章节可跳读。
- 风格定位：
  - 不写成教科书，不堆砌概念
  - 强调「能复制的步骤」「实战工作流」「避坑经验」
  - 多用图示、示例、对照表，少用长篇大论

## 结构偏好

- 根目录保持简洁：`README.md`、`docs/`、`drafts/`、`MEMORY.md`。
- 正式教程全部放在 `docs/` 下，按章节划分子目录：
  - `docs/ch01-quickstart`
  - `docs/ch02-concepts`
  - `docs/ch03-playbook`
  - 后续章节沿用类似命名（`ch04-...`）。
- 旧稿、研究资料、长文草稿统一归档到 `drafts/`，只做参考，不直接对外暴露给读者。

## 内容取舍原则

- 正文尽量短：优先给出决策、步骤和示例，把长篇论证放在附录或草稿。
- 所有「版本敏感信息」（模型版本、价格、API 细节等）集中维护在专门的表格 / 附录文件，避免频繁修改正文。
- 实战技巧优先：
  - 先写「今天就能用起来」的章节（快速上手、实战 Playbook）
  - 再逐步补充原理与架构类内容

## 工具与产品范围

- 主线工具：Claude Code、Codex（本仓库所在环境）、Cursor。
- 对照与补充：GitHub Copilot、Cline、Trae 等。
- 若需加入新工具，请在总览 README 中更新「适用读者 / 工具列表」说明。

## Agent 协作偏好

- 当 Agent 修改结构时：
  - 优先保持 `drafts/` 中历史内容完整，不覆盖、不删除。
  - 对公开文档（`docs/`、`README.md`）做结构性改动时，在提交信息中简要说明动机。
- 当内容存在取舍冲突时：
  - 优先保证「新读者第一次打开仓库时的体验」，即 `README.md` 的清晰程度。
