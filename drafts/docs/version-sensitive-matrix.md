## 五、更新与维护建议

- 建议把本文件当作“内部工作文档 + 附录数据源”，不直接在正文复制其中表格；  
- 每次对这里的数据做较大调整时，在 Git 提交信息中简要说明原因（如“根据 Anthropic 官方价格页更新 Claude 系列档位”）；  
- 教程对外发布前，集中做一次“版本敏感信息体检”：  
  - 检查所有正文中的具体版本号/价格是否只存在于附录或本文件；  
  - 检查引用的官方链接是否仍然可访问且不过期；  
  - 在附录中注明“校验日期”和“下一次建议复查时间”（例如每季度一次）。
