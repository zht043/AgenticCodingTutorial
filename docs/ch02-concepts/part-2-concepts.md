# Chapter 2 · Agent 运作原理与相关基本概念

> 目标：用尽量少的概念，帮你搞清楚 Agent 在背后到底做了什么。

本章会围绕一个你已经跑通的 Agent 实例，解释它的「工作原理」，但我们只保留**和日常开发强相关**的部分。

## 1. 从一次对话看完整执行链路

我们会从一个具体例子出发，比如：

> 你在编辑器里选中一段代码，对 Agent 说：「帮我重构成更易读的版本并补上测试。」

然后拆解 Agent 背后的步骤：

- 如何把你的自然语言需求和代码上下文打包成请求
- 模型如何基于上下文「思考」和生成结果
- Agent 如何决定是否需要调用额外工具（文件系统、终端、测试命令等）
- 最终生成的变更是如何回到你的代码里的

这一节会配一张「从输入到落地代码」的时序/流程图。

## 2. 几个必须搞清楚的概念

本节会用开发者视角解释这些关键词：

- **模型（Model）**：真正「在脑子里算东西」的部分
- **Agent**：帮你管理上下文、调用工具、规划步骤的「调度层」
- **Tools / MCP / Skills / 插件**：让 Agent 能够「读文件、跑命令、调接口」的扩展能力
- **Memory / 上下文管理**：Agent 如何记住你的项目、偏好和当前任务

每个概念都会有：一句话定义 + 一个开发者能共情的例子 + 一两个常见坑。

## 3. 为什么 Agent 有时很聪明，有时又很蠢？

我们会讨论：

- 上下文窗口、提示词质量、代码结构，对效果的影响
- 「可观测性」：你能看到 Agent 做了什么吗？
- 什么时候可以一键接受建议，什么时候必须手动审查

这一节会给出一套「心智模型」，让你对 Agent 的期待更匹配现实，从而用对它。

## 4. 模型 / 工具 / 产品之间的关系

很多人会混淆：Claude / GPT / DeepSeek / Claude Code / Cursor / Copilot ……

本节会用一个简单的对照表说明：

- 模型厂商 vs 开发者工具 vs 编辑器插件
- 哪些东西可以替换，哪些是「生态」的选择
- 如何根据你所在的团队 / 项目做最小成本的选择
